### 工具选择

#### Hexo

凭着自己感觉写了，也不copy网上的长篇大论了。Hexo说白了就是搭建个人博客的一个框架，同类的还是一个叫 Jekyll，一开始尝试用Jekyll，mac安装相当麻烦，博主是个懒汉，就在网上继续找啊找，网友说Hexo比Jekyll好，我猜也是，就安装了。

<!--more-->
##### 步骤

由于博主之前有GitHub,关于GitHub相关的步骤我就不在这里吹逼了，小伙伴们可以自行去搜。另外一个就是要安装node,也是自行百度，因为写博客的时候博主已经忘了node是怎么安装到我mac上的－－。


安装hexo:

```
sudo npm install -g hexo
```

初始化工程：

```
hexo init 
```

初始化完毕进入init执行的路径下，你就会发现很神奇，然后

```
hexo generate
```

这就生成了静态页面，然后

```
hexo server
```

服务启动，访问http://localhost:4000


简直爆炸，你会看到一个很炫很牛逼的页面，基本hexo环境就搭建完毕了。


##### 小结

如果还没有用过或者没有自己GitHub仓库的同学建议还是先熟悉熟悉Git，然后再搭建一个基于GitHub的博客。安装node可能会遇到很多问题，一般在网上搜解决方案都能解决。个人觉得hexo还是很简单的，至少比Jekyll方便一些。


### 搭建

说完了hexo的简单用法，接下来就是重头戏了，吹逼半天怎么搭建博客？


#### 教程

之前博主都是在开源中国或者其他博客论坛上写一些博客，并没有关注很多内容之外的细节，这次突发奇想自己搭博客时，也没有意识到其实个人博客可以根据自己的喜好设计专属自己的博客网站，每个页面都可以很个性，直到开始搭博客并按照这位博主的博客地址http://www.devchen.com/ 以及教程 https://github.com/sykiechen/hexo-theme-hic17/ 按照上面的修改点一点一点地迭代自己的博客，才发现原来这么神奇。

#### 总结与心得

根据上面这位博主的教程基本上就能搭建出自己比较满意的页面，不过算是一种傻瓜式教程，基本上在哪一行改什么代码都写的清清楚楚，但是作为有梦想的博客主们，如果对于页面还有其他的一些想法，可以参见我的这篇总结,基本有以下几点

##### theme主题
    
hexo默认的是Landscape，我使用的也是，如果各位觉得这个默认主题不够好，完全可以自行去下载新的主题模版，然后统一在_config.yml中配置，我当时也搜索过，发现推荐最多的就是这篇文章，http://www.zhihu.com/question/24422335/ 主题说白了就是一个大体的结构，结构有了具体填充就由各位自行想象了。

##### 界面美化与功能性丰富

如字体修改，页面 页首 页脚展示修改都属于界面美化，一般都在source/css or source/js 下修改，以及一些功能性丰富，如：微博分享，各种社交网站图标以及分享功能，都在Font Awesome（http://fontawesome.io/）, 这个网站中找到，一般下载最新版该有的都有，第三方社交平台也都会有开发者中心&开放者平台，api都很详细，自己有账号一看就知道怎么弄了，加到你的网站上so beautiful.

##### 页面加载优化

一般都是由于hexo提供的Landscape等主题js/css加载的都是google服务，然后你懂的，不翻墙呵呵。不过如果部署在GitHub上有可能google反而快点，另外，根据博主的试验，强烈建议不要修改layout\_partial\head.ejs 第31行fonts.googleapis.com这个地址，fonts.useso.com更慢。。。总之，我的建议，一个一个地方改，改完都deploy一下看效果，不要图快。
 
##### 第三方工具引入 

看到博主的微博窗，评论模块还有站长统计了吗，熟悉博客搭建的同学肯定都清楚，这是博客最基本的一些工具，不过第一次玩博客的同学真的不一定清楚，我详细介绍下，像博主我使用的评论就是“多说”平台，这个平台功能还是挺强大的，相当于你博客评论的服务端，提供前端插件的同时，还可以对博客文章的评论做各种限制功能。统计方面用的是umeng,博主作为做push服务的服务端，对友盟还是有所了解的，虽然不清楚他们的push做的如何，但至少统计这块做的还是很66的，最后就是微博秀，没什么好说的了，新浪提供的服务，网址都在这，大家自行去研究吧！

##### 个性化域名

最后提一下个性化域名的问题，博主在搭建博客初期没有准备用个性化域名，因为博主没钱！！！。哈哈，开玩笑，其实是觉得没必要，因为用GitHub的域名我觉得就ok了，因为毕竟大家都不是很关注访问什么url，而是里面的内容是否感兴趣。但是当我搭建完毕才发现，Github的网站协议是Https了，这就坑爹了，deploy到上面后发现有些js居然被认为是不安全的脚本（如微博秀，返回顶部），加载不到页面去，还需要手动点（当然和浏览器有关，chrome有这个问题，Safari没这个问题），特别不友好，不明真相的以为博客有什么bug呢，，肯定用户流失特别大。所以狠下心买域名，我用的是阿里云
https://wanwang.aliyun.com/ 买了个29一年的先玩玩，现在有点坑，要实名认证，所以买完不能立刻用，一般第二天就ok了，直接在后台配置CNAME就好，指向Github的域名。这个时候博主发现一个比较坑的问题了，原来Github搭建博客的时候一定要把仓库起名为username.github.io，否则在配置config_yml文件的url时 配root /时访问首页发现，css js都加载不了。。。而且用买的域名访问也被404，所以一定要注意这个问题。

最后总结一下
(1)搭建GitHub仓库的时候就用username.github.io作为个人主站，而不是某一个项目的博客主页。
(2)遇到css.js加载不了一般都是路径问题，仔细确认下。
(3)遇到问题发现不了原因而且搞不定的时候可以看看其他个人博客的代码或者配置，就能找到异同。
